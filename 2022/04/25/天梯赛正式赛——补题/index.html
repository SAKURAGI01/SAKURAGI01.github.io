    L1 - 3 谁能进图书馆

    题目描述

    为了保障安静的阅读环境，有些公共图书馆对儿童入馆做出了限制。例如“12 岁以下儿童禁止入馆，除非有 18 岁以上（包括 18 岁）的成人陪同”。现在有两位小/大朋友跑来问你，他们能不能进去？请你写个程序自动给他们一个回复。
    输入格式：
    输入在一行中给出 4 个整数：
    禁入年龄线 陪同年龄线 询问者1的年龄 询问者2的年龄
    这里的禁入年龄线是指严格小于该年龄的儿童禁止入馆；陪同年龄线是指大于等于该年龄的人士可以陪同儿童入馆。默认两个询问者的编号依次分别为 1 和 2；年龄和年龄线都是 [1, 200] 区间内的整数，并且保证 陪同年龄线 严格大于 禁入年龄线。
    输出格式：
    在一行中输出对两位询问者的回答，如果可以进就输出 年龄-Y，否则输出 年龄-N，中间空 1 格，行首尾不得有多余空格。
    在第二行根据两个询问者的情况输出一句话：
    如果两个人必须一起进，则输出 qing X zhao gu hao Y，其中 X 是陪同人的编号， Y 是小孩子的编号；
    如果两个人都可以进但不是必须一起的，则输出 huan ying ru guan；
    如果两个人都进不去，则输出 zhang da zai lai ba；
    如果一个人能进一个不能，则输出 X: huan ying ru guan，其中 X 是可以入馆的那个人的编号。

    输入

    在一行中输出对两位询问者的回答，如果可以进就输出 年龄-Y，否则输出 年龄-N，中间空 1 格，行首尾不得有多余空格。

    输出

    如果两个人必须一起进，则输出 qing X zhao gu hao Y，其中 X 是陪同人的编号， Y 是小孩子的编号；
    如果两个人都可以进但不是必须一起的，则输出 huan ying ru guan；
    如果两个人都进不去，则输出 zhang da zai lai ba；
    如果一个人能进一个不能，则输出 X: huan ying ru guan，其中 X 是可以入馆的那个人的编号。

    样例

    样例

    Example 1

    Input

    12 18 18 8

    Output

    18-Y 8-Y
qing 1 zhao gu hao 2

    Example 2

    Input

    12 18 10 15

    Output

    10-N 15-Y
2: huan ying ru guan

    注意

    第一批可以选择 175、189、203、256 这四个分数的学生各一名，此外 175 分 PAT 分数达到 90 分的学生和 189 分 PAT 分数达到 95 分的学生可以额外进入名单。第二批就只剩下 175、189 两个分数的学生各一名可以进入名单了。最终一共 8 人进入推荐名单。

    解题思路

    这题的关键是理清楚几种分类讨论，把二人的年龄段分为三份，并在草稿纸上列出一共有九种情况，注意情况的合并即可

    关键还要注意N和Y的输出顺序，才能那道满分！

    AC代码

    #include<bits/stdc++.h>
using namespace std;
int main(){
    int a,b,c,d;
    cin>>a>>b>>c>>d;
    int x,y;
    if(c<a){
        if(d<a){
            cout<<c<<"-N "<<d<<"-N"<<endl;
            cout<<"zhang da zai lai ba"<<endl;
        }
        else if(d>=a&&d<b){
            cout<<c<<"-N "<<d<<"-Y"<<endl;
            cout<<"2: huan ying ru guan";
        }
        else{
            cout<<c<<"-Y "<<d<<"-Y"<<endl;
            cout<<"qing 2 zhao gu hao 1";
        }
    }else if(c>=b){
        if(d<a){
            cout<<c<<"-Y "<<d<<"-Y"<<endl;
            cout<<"qing 1 zhao gu hao 2";
        }
        else if(d>=a){
              cout<<c<<"-Y "<<d<<"-Y"<<endl;
              cout<<"huan ying ru guan";
        }
    }else{
        if(d<a){
            cout<<c<<"-Y "<<d<<"-N"<<endl;
            cout<<"1: huan ying ru guan";
        }else if(d>=a){
            cout<<c<<"-Y "<<d<<"-Y"<<endl;
             cout<<"huan ying ru guan";
        }
    }
}

    L1 - 8 静静的推荐

    题目描述

    天梯赛结束后，某企业的人力资源部希望组委会能推荐一批优秀的学生，这个整理推荐名单的任务就由静静姐负责。企业接受推荐的流程是这样的：
    只考虑得分不低于 175 分的学生；
    一共接受 K 批次的推荐名单；
    同一批推荐名单上的学生的成绩原则上应严格递增；
    如果有的学生天梯赛成绩虽然与前一个人相同，但其参加过 PAT 考试，且成绩达到了该企业的面试分数线，则也可以接受。
    给定全体参赛学生的成绩和他们的 PAT 考试成绩，请你帮静静姐算一算，她最多能向企业推荐多少学生？

    输入

    输入第一行给出 3 个正整数：N（≤105）为参赛学生人数，K（≤5×103）为企业接受的推荐批次，S（≤100）为该企业的 PAT 面试分数线。
    随后 N 行，每行给出两个分数，依次为一位学生的天梯赛分数（最高分 290）和 PAT 分数（最高分 100）。

    输出

    在一行中输出静静姐最多能向企业推荐的学生人数。

    样例

    Input

    10 2 90
203 0
169 91
175 88
175 0
175 90
189 0
189 0
189 95
189 89
256 100

    Output

    8

    注意

    第一批可以选择 175、189、203、256 这四个分数的学生各一名，此外 175 分 PAT 分数达到 90 分的学生和 189 分 PAT 分数达到 95 分的学生可以额外进入名单。第二批就只剩下 175、189 两个分数的学生各一名可以进入名单了。最终一共 8 人进入推荐名单。

    解题思路

    这道题乍一看，挺难理解的，主要顺出来两种情况

    a. 满足175要求的，PAT满足要求的特批进入

    b. 其它的175的，按批次进入

    知识点：桶排序

    AC代码

    #include <iostream>
#include <cstring>

using namespace std;

const int N = 1e5 + 10;

int b[N];

bool st[N];

struct Stu
{
    int x, y;
}a[N];

bool cmp(Stu a, Stu b)
{
    if (a.x == b.x) return a.y > b.y;
    return a.x > b.y;
}


int main()
{
    int n, k, s;
    cin >> n >> k >> s;

    int sum = 0;
    for (int i = 0; i < n; i ++ )
    {
        cin >> a[i].x >> a[i].y;
        if (a[i].x >= 175 && a[i].y >= s)
            st[i] = true, sum ++ ;
    }

    for (int i = 0; i < n; i ++ )
        if (!st[i]) b[a[i].x] ++ ;

    for (int i = 175; i <= 290; i ++ )
    {
        if (b[i] > k) sum += k;
        else sum += b[i];
    }

    cout << sum << endl;

    return 0;
}

    L2 - 1 插松枝

    题目描述

    人造松枝加工场的工人需要将各种尺寸的塑料松针插到松枝干上，做成大大小小的松枝。他们的工作流程（并不）是这样的：
    每人手边有一只小盒子，初始状态为空。
    每人面前有用不完的松枝干和一个推送器，每次推送一片随机型号的松针片。
    工人首先捡起一根空的松枝干，从小盒子里摸出最上面的一片松针 —— 如果小盒子是空的，就从推送器上取一片松针。将这片松针插到枝干的最下面。
    工人在插后面的松针时，需要保证，每一步插到一根非空松枝干上的松针片，不能比前一步插上的松针片大。如果小盒子中最上面的松针满足要求，就取之插好；否则去推送器上取一片。如果推送器上拿到的仍然不满足要求，就把拿到的这片堆放到小盒子里，继续去推送器上取下一片。注意这里假设小盒子里的松针片是按放入的顺序堆叠起来的，工人每次只能取出最上面（即最后放入）的一片。
    当下列三种情况之一发生时，工人会结束手里的松枝制作，开始做下一个：
    （1）小盒子已经满了，但推送器上取到的松针仍然不满足要求。此时将手中的松枝放到成品篮里，推送器上取到的松针压回推送器，开始下一根松枝的制作。
    （2）小盒子中最上面的松针不满足要求，但推送器上已经没有松针了。此时将手中的松枝放到成品篮里，开始下一根松枝的制作。
    （3）手中的松枝干上已经插满了松针，将之放到成品篮里，开始下一根松枝的制作。
    现在给定推送器上顺序传过来的 N 片松针的大小，以及小盒子和松枝的容量，请你编写程序自动列出每根成品松枝的信息。

    输入

    输入在第一行中给出 3 个正整数：N（≤103），为推送器上松针片的数量；M（≤20）为小盒子能存放的松针片的最大数量；K（≤5）为一根松枝干上能插的松针片的最大数量。
    随后一行给出 N 个不超过 100 的正整数，为推送器上顺序推出的松针片的大小。

    输出

    每支松枝成品的信息占一行，顺序给出自底向上每片松针的大小。数字间以 1 个空格分隔，行首尾不得有多余空格。

    样例

    Input

    8 3 4
20 25 15 18 20 18 8 5

    Output

    20 15
20 18 18 8
25 5

    解题思路

    当时，没有考虑几种特殊情况，最后没完全写出来。

    这个题意也比较复杂，要将插松针的过程在草稿纸上模拟出来，再去构思

    这里我用数组模拟栈和队列，

    主要要弄清楚两点：

    1、题意比较复杂，总的来说就是三个输出条件，满足任意一个就可以输出

    1）盒子中的不满足，推送器没松针片了

    2）小盒子满了

    3）松针干上插满了

    2、什么时候可以插松针干？

    先从盒子中找，再在推送器上找的，且必须满足松针片要小于前一个

    AC代码

    #include <bits/stdc++.h>

using namespace std;

int a[1005], b[1005], in[5005], bs, as;

// a中存放的是成品篮的数据，in中存放的是推送器
// as是松针干上的松针片数量，bs是盒子中放松针片的数量

void ok()
{
    if (!as)
        return; //如果松针干上的松针片数量等于0

    for (int i = 1; i <= as; i++)
    {
        printf("%d", a[i]);
        if (i != as)
            putchar(' ');
    }

    as = 0, puts("");
}

int main()
{
    int n, m, k, x, s = 0;
    cin >> n >> m >> k;
    a[0] = 999999999;

    for (int i = 1; i <= n; i++)
        cin >> in[i];

    for (int i = 1; i <= n + n; i++)
    {
        if (as == k)
            ok(); //松针干上插满松针

        if (b[bs] <= a[as] && bs > 0)
            a[++as] = b[bs--]; //当盒子中的松针片小于松针干上的就放到松针干上
        else
        {
            x = in[++s]; // x是现在要取的松针片
            if (s > n)
                x = 1e9; // s是推送器上的顺序

            if (x <= a[as])
                a[++as] = x; //当现在取的松针片小于松针干上的松针片时
            else if (bs == m)
                s--, ok(); //盒子放满了
            else if (x < 1e9)
                b[++bs] = x; //放在盒子里
            else
                ok(); //不满足要求且没有松针
        }
    }
}

    天梯赛总结

    ​ 这次的比赛不是那么尽如人意，还是平时训练得太少了，还是要多刷题，才能在比赛的时候真正熟练地想到相应的知识点。

    ​ 我以后还是要合理安排好比赛和专业课学习的时间，不能顾此失彼，每周要至少两场比赛的训练，还要至少掌握一个类型的算法题，这样积少成多，多加训练，逐渐就能做出来中等偏上难度的题目。

    ​ 还有，现在即将要来的是icpc省赛，和队友的配合也需要在后面进行队内训练，要点如下:

    1. 注意随时看榜单

    对于我们这些初学者，肯定不会最早的发现签到题拿一血，所以榜单是至关重要的，这样有助于我们快速进入写题状态，并切掉签到题，而且，在后期可以从榜单上发现，哪些题目是可以争取的，是取得好成绩的关键。

    1. 和队友有效分工

    “有效”两个字是最关键的，对于签到题，可以分开做，比如，三个人其中一个会写，另外两个可以去审其他题目，但是，如果那个人做不出来了，或者超时了，就不能放任不管，两个人一定要回来继续想那道题目

    1. 五个小时，不长不短，坚持到最后

    记住：

    不是一个人的王者,而是团队的荣耀 ——出自王者荣耀
